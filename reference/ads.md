
# 광고관리 페이지 고도화

## 데이터 필터링 추가

- AdStatusCards 의 위에 광고 데이터 필터링 조건을 덧붙인다.
- 광고 데이터 필터링 조건은 크게 세가지
  - 총판
  - 대행사
  - 광고주
- 필터링은 드롭리스트로 보여진다. 
  - 왼쪽은 총판 리스트
  - 중간은 대행사 리스트
  - 오른쪽은 광고주 리스트
- 대행사 리스트는 총판에 속하는 대행사들만 보여야한다.
- 광고주 리스트는 대행사에 속하는 광고주들만 보여야한다.
- 대행사 리스트는 총판 선택에 따라 실시간으로 연동되어야한다.
  - 총판 선택에 따라 대행사의 '전체'가 영향을 받는다.
  - 해당 총판에 소속된 organization 들만 보여야함
- 광고주 리스트는 대행사 선택에 따라 실시간 연동되어야한다.
  - 대행사 선택에 따라 광고주의 '전체'가 영향을 받는다.
  - 해당 대행사에 소속된 광고주들만 보여야한다.
- 모든 드롭리스트는 기본적으로 전체가 항상 선택되어있다.
  - 전체가 선택되어있으므로 광고도 모든 광고를 가져온다.
- 광고 페이지에 보이는 데이터는 각 필터링 조건에 알맞는 광고들만 가져와야한다.
  - 조건: 총판, 대행사, 광고주
- 사용자 role이 총판일 경우 드롭리스트는 총판, 대행사, 광고자 세개 다 보인다.
- 사용자 role이 대행사일 경우 드롭리스트는 광고주만 보인다.
- 사용자 role이 광고주일 경우 드롭리스트는 보이지 않는다.
  - 사용자 role이 광고주일 경우 항상 본인의 광고만 가져온다. (기존 구현과 동일)


## 예시
### 전제
총판은 루카스, 데이브, 올리브 이렇게 세 종류가 있다.

루카스의 대행사(조직)는 알파, 베타 이렇게 두 종류
데이브의 대행사(조직)은 감마
올리브의 대행사(조직)은 세타, 제타 이렇게 두 종류

알파의 광고주는 yellow, green 이렇게 두 종류
베타의 광고주는 red, orange 이렇게 두 종류
감마의 광고주는 blue
세타의 광고주는 없음
제타의 광고주는 stone

### 초기 UI
- 총판 리스트에서는 전체, 루카스, 데이브, 올리브가 보여야한다.
- 대행사 리스트에서는 전체, 알파, 베타, 감마, 세타, 제타가 보여야한다.
- 광고주 리스트에서는 전체, yellow, green, red, orange, blue, stone이 보여야한다.

### 만약 총판 리스트에서 루카스를 골랐을 경우
- 대행사 리스트에서는 전체, 알파, 베타가 보여야한다.
- 광고주 리스트에서는 전체, yellow, green, red, orange 가 보여야한다.

### 만약 대행사 리스트에서 알파를 골랐을 경우
- 광고주 리스트에서는 전체, yellow, green 가 보여야 한다.

### 다시 총판 리스트에서 데이브를 골랐을 경우
- 대행사 리스트에서는 전체, 감마가 보여야한다.
- 광고주 리스트에서는 blue가 보여야한다.
